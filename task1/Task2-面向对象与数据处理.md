## 练习任务

### 1. RoboMaster 队员积分管理系统

RoboMaster 战队日常运营中，经常需要记录队员的出勤、任务完成情况并统计积分。Task-1 中你用过程式编程完成了猜数游戏，现在请你用**面向对象**的方式，编写一个**命令行队员积分管理系统**。

> **和 Task-1 的区别**：Task-1 重在"逻辑"（if / while / 函数），Task-2 重在"设计"（怎么用类来组织代码）。同一个功能，用 OOP 写出来的代码结构是截然不同的。

#### 功能需求

程序启动后显示一个菜单，支持以下操作：

```
========== RoboMaster 队员积分管理系统 ==========
```

**各功能说明：**

| 功能         | 说明                                                                         |
| ---------- | -------------------------------------------------------------------------- |
| **添加队员**   | 输入姓名、组别（视觉/电控/机械/运营），初始积分默认为 0，自动生成队员编号。编号规则：`RM` + 四位数字（如 `RM0001`），自动递增。 |
| **查看所有队员** | 以表格形式展示所有队员信息：编号、姓名、组别、积分。                                                 |

| **为队员扣分**  | 输入队员编号和分数（正数），减少该队员的积分。                                                    |
| **按积分排名**  | 从高到低展示队员积分排行榜。                                                             |
| **删除队员**   | 输入队员编号，删除该队员（需二次确认）。                                                       |
| **退出系统**   | 退出程序。**注意**：本任务暂不要求保存数据，退出后数据会丢失（Task-3 将学习如何持久化存储）。                       |

#### 设计要求

1. **属性和方法**：每个类必须用 `__init__` 初始化属性，方法通过 `self` 访问属性。
2. **多文件拆分**：`Member` 和 `MemberManager` 分属不同的 `.py` 文件，`main.py` 通过 `import` 导入使用。不允许把所有代码写在一个文件里。
3. **输入校验**：对用户输入做基本的合法性检查（如编号是否存在、分数是否为正数等）。
4. **队员编号自增**：`next_id` 应使用**类属性**实现自动递增，而非在 `main.py` 中手动维护编号。
5. **`__str__`** **魔法方法**：为 `Member` 类实现 `__str__`，使 `print(member)` 时能友好地展示队员信息。

#### 附加功能（选做，加分项）

以下功能不强制，实现任意一项可获得额外评价：

- **搜索队员**：按姓名或组别模糊搜索队员。
- **修改队员信息**：修改队员的姓名或组别。
- **数据统计**：统计各组别的人数、平均积分、最高积分等。
- **防止重复添加**：同一姓名不能重复添加为队员。

### 2. 知识点清单

在开始写代码之前，建议先掌握以下知识点：

| 类别                            | 知识点                                                 | 在项目中的应用                      |
| ----------------------------- | --------------------------------------------------- | ---------------------------- |
| **类与对象**                      | `class` 关键字、`__init__` 构造方法、`self` 参数               | 定义 Member、MemberManager 类    |
| **实例方法**                      | 方法的定义与调用、通过 `self` 访问属性                             | `add_score()`、`rank_all()` 等 |
| **实例属性 vs 类属性**               | 实例属性在 `__init__` 中定义，类属性直接在类体中定义                    | `name` 是实例属性，`next_id` 是类属性  |
| **魔法方法**                      | `__str__`                                           | 美化 `print(member)` 的输出       |
| **模块与导入**                     | `import`、`from ... import`、`__name__ == "__main__"` | 拆分代码到多个 `.py` 文件             |
| **列表与字典**                     | 列表存储对象、遍历、排序；字典的增删改查                                | `members` 列表管理所有队员           |
| **`sort`** **与** **`lambda`** | `list.sort(key=lambda x: x.score, reverse=True)`    | 积分排名                         |
| **输入校验**                      | `try/except`、`strip()`、条件判断                         | 防止无效输入导致程序崩溃                 |
